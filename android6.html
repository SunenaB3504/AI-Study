Data Storage Overview

Android provides several options for storing data: SharedPreferences, Internal/External Storage, and databases like SQLite and Room.

Choose the appropriate storage method based on your data type, size, and access patterns.

SharedPreferences

Key-value pairs for small amounts of primitive data

  • Simple key-value storage
  • Private to the app
  • Not suitable for large data

Internal Storage

Private files stored on device memory

  • Always available
  • Automatically deleted on uninstall
  • Private to the app

External Storage

Files stored on shared/external storage

  • Can be removed by user
  • Requires permissions
  • Visible to other apps

SharedPreferences

SharedPreferences is Android's lightweight mechanism for storing small amounts of key-value data. It's perfect for storing user preferences, settings, and small configuration data.

Basic SharedPreferences Usage

// Get SharedPreferences instance
SharedPreferences prefs = getSharedPreferences("MyPrefs", MODE_PRIVATE);

// Create editor to modify preferences
SharedPreferences.Editor editor = prefs.edit();

// Store data
editor.putString("username", "sunil");
editor.putInt("user_age", 25);
editor.putBoolean("notifications_enabled", true);

// Apply changes
editor.apply(); // Asynchronous
// or editor.commit(); // Synchronous

// Retrieve data
String username = prefs.getString("username", "default_value");
int age = prefs.getInt("user_age", 0);
boolean notifications = prefs.getBoolean("notifications_enabled", false);

✅ Advantages

  • Simple to use
  • Fast access to data
  • Thread-safe operations
  • Automatic data persistence

⚠️ Limitations

  • Not suitable for large data
  • No complex queries
  • Only primitive data types
  • Not encrypted by default

Internal Storage

Internal storage provides private file storage that's accessible only to your app. Files are automatically deleted when the app is uninstalled.

Writing to Internal Storage

String filename = "myfile.txt";
String data = "Hello, Internal Storage!";

try {
    FileOutputStream fos = openFileOutput(filename, Context.MODE_PRIVATE);
    fos.write(data.getBytes());
    fos.close();
    Toast.makeText(this, "File saved", Toast.LENGTH_SHORT).show();
} catch (Exception e) {
    e.printStackTrace();
}

Reading from Internal Storage

String filename = "myfile.txt";

try {
    FileInputStream fis = openFileInput(filename);
    StringBuilder sb = new StringBuilder();
    int ch;
    while ((ch = fis.read()) != -1) {
        sb.append((char) ch);
    }
    fis.close();
    String content = sb.toString();
    Toast.makeText(this, content, Toast.LENGTH_SHORT).show();
} catch (Exception e) {
    e.printStackTrace();
}

File Access Modes

MODE_PRIVATE

File can only be accessed by the calling app

MODE_APPEND

Append to existing file instead of overwriting

External Storage

External storage refers to shared storage that can be accessed by all apps and the user. It requires proper permissions and should be used carefully.

Writing to External Storage

// Check permission
if (ContextCompat.checkSelfPermission(this, Manifest.permission.WRITE_EXTERNAL_STORAGE)
    != PackageManager.PERMISSION_GRANTED) {
    ActivityCompat.requestPermissions(this,
        new String[]{Manifest.permission.WRITE_EXTERNAL_STORAGE},
        REQUEST_WRITE_PERMISSION);
}

// Write file
File file = new File(getExternalFilesDir(null), "data.txt");
try {
    FileOutputStream fos = new FileOutputStream(file);
    fos.write("Hello, External Storage!".getBytes());
    fos.close();
} catch (IOException e) {
    e.printStackTrace();
}

📱 Permission Requirements

  • Android 6.0+: Request WRITE_EXTERNAL_STORAGE at runtime
  • Android 10+: Use scoped storage for better privacy
  • Android 11+: No permission needed for app-specific directory

SQLite Database

SQLite is a lightweight, embedded SQL database engine that's included with Android. It's suitable for structured data that requires complex queries.

Basic SQLite Operations

// Open or create database
SQLiteDatabase db = openOrCreateDatabase("mydb", MODE_PRIVATE, null);

// Create table
db.execSQL("CREATE TABLE IF NOT EXISTS users(id INTEGER PRIMARY KEY, name TEXT, email TEXT)");

// Insert data
db.execSQL("INSERT INTO users VALUES(1, 'Sunil', 'sunil@example.com')");

// Query data
Cursor cursor = db.rawQuery("SELECT * FROM users", null);
if (cursor.moveToFirst()) {
    do {
        String name = cursor.getString(cursor.getColumnIndex("name"));
        String email = cursor.getString(cursor.getColumnIndex("email"));
        // Process data
    } while (cursor.moveToNext());
}
cursor.close();

// Close database
db.close();

SQLite Features

  • ACID compliance
  • SQL query support
  • Transactions
  • Views and triggers
  • Small footprint
  • Cross-platform
  • No configuration needed
  • Built into Android

Room Database (Recommended)

Room is Google's recommended persistence library for Android. It provides an abstraction layer over SQLite that makes database operations easier and more robust.

Entity

Represents a database table

DAO

Data Access Object for queries

Database

Main database holder

Room Entity

@Entity(tableName = "users")
public class User {
    @PrimaryKey(autoGenerate = true)
    public int id;

    @ColumnInfo(name = "full_name")
    public String name;

    @ColumnInfo(name = "user_email")
    public String email;

    public User(String name, String email) {
        this.name = name;
        this.email = email;
    }
}

Room DAO

@Dao
public interface UserDao {
    @Insert
    void insert(User user);

    @Update
    void update(User user);

    @Delete
    void delete(User user);

    @Query("SELECT * FROM users ORDER BY name ASC")
    List<User> getAllUsers();

    @Query("SELECT * FROM users WHERE id = :userId")
    User getUserById(int userId);
}

Room Database

@Database(entities = {User.class}, version = 1, exportSchema = false)
public abstract class AppDatabase extends RoomDatabase {
    public abstract UserDao userDao();

    private static volatile AppDatabase INSTANCE;

    public static AppDatabase getDatabase(final Context context) {
        if (INSTANCE == null) {
            synchronized (AppDatabase.class) {
                if (INSTANCE == null) {
                    INSTANCE = Room.databaseBuilder(context.getApplicationContext(),
                            AppDatabase.class, "user_database")
                            .build();
                }
            }
        }
        return INSTANCE;
    }
}

Why Use Room?

  • Compile-time SQL validation
  • Reduces boilerplate code
  • Built-in threading support
  • Easy testing with Hilt/Dagger
  • LiveData and Flow integration
