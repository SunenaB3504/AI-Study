1.1 A Little Background

The Not So Distant Past

Historically, mobile phones were closed environments built on proprietary operating systems that restricted third-party applications. This created artificial barriers for developers hoping to build on increasingly powerful mobile hardware.

Early mobile development required low-level C or C++ programming with heavy use of proprietary APIs that were difficult to use. This difficulty was amplified when developing applications that needed to work across different hardware implementations.

The Future

Android sits alongside a new wave of mobile operating systems designed for increasingly powerful mobile hardware. Unlike many proprietary platforms, Android offers an open development environment built on an open source Linux kernel.

1.2 An Open Platform for Mobile Development

Google describes Android as "the first truly open and comprehensive platform for mobile devices." Android is made up of several necessary and dependent parts that work together to create a complete mobile development platform.

Android Components

Hardware Reference Design

Describes the capabilities required of a mobile device to support the Android software stack, including specifications for processors, memory, storage, and peripherals.

Linux Operating System

Core services including hardware drivers, process and memory management, security, network, and power management, all optimized for mobile devices.

Open Source Libraries

C/C++ core libraries including SQLite, WebKit, OpenGL, and SSL for application development.

Android Runtime

Core libraries and Dalvik virtual machine that power Android applications.

What Makes Android Special

Unlike other mobile platforms, Android doesn't differentiate between native applications and those developed by third parties. All applications have equal standing and access to the same underlying services and hardware.

Key Android Features

  • ‚Ä¢ Complete hardware access for all applications

Application Equality

Android doesn't distinguish between native and third-party applications. Your applications can replace native applications and have access to the same data and hardware resources. This means you can create a custom home screen, dialer, or any other system component.

1.4 Android SDK Features

The Android SDK provides powerful APIs that give developers access to the full range of Android capabilities. Here are some of the most noteworthy features available to developers.

Core Android APIs

Hardware Access

Comprehensive APIs for accessing device hardware including camera, GPS, accelerometer, compass, and other sensors.

Key Features: Camera control, GPS location, motion sensors, device orientation, hardware status monitoring
Location-Based Services

Native support for Google Maps with full programmatic control, geocoding, and location-based services using GPS and cell tower triangulation.

Key Features: Interactive maps, route planning, location tracking, proximity alerts, address lookup
Data Management

SQLite database for structured data storage, content providers for data sharing, and preferences for simple data persistence.

Key Features: Local database storage, data synchronization, inter-application data sharing, backup and restore
Background Processing

Services for background processing, notifications for user alerts, and alarms for scheduled tasks.

Key Features: Background services, scheduled tasks, user notifications, system event handling
Communication

APIs for SMS, instant messaging, email, and peer-to-peer communication using Google Talk.

Key Features: SMS sending/receiving, instant messaging, email integration, Bluetooth communication
Media & Graphics

Comprehensive media support including audio/video playback and recording, 2D/3D graphics with OpenGL ES, and image processing.

Key Features: Audio/video playback, camera integration, 3D graphics, image processing, animation frameworks

1.6 Why Develop for Android?

Android represents a unique opportunity for developers. With its open platform, comprehensive SDK, and zero development costs, Android makes mobile development accessible to everyone.

Developer Advantages

No Barriers to Entry

No licensing fees, no development costs, no certification requirements. Anyone with programming skills can start developing Android applications immediately.

Equal Opportunity

Third-party applications have the same access to hardware and data as native applications. Your apps can replace or extend any system component.

Rich Feature Set

Access to GPS, camera, accelerometer, maps, background services, databases, and comprehensive communication APIs.

Growing Market

Android's market share is growing rapidly with millions of devices in use worldwide. The Android Marketplace provides global distribution for your applications.

What Android Has That Others Don't
  • ‚Ä¢ Native Google Maps: Full programmatic control of Google Maps in your applications
  • ‚Ä¢ Background Services: True multitasking with services that run invisibly
  • ‚Ä¢ Application Equality: No distinction between native and third-party apps
  • ‚Ä¢ P2P Messaging: Built-in peer-to-peer communication using Google Talk
  • ‚Ä¢ Open Philosophy: Complete access to system internals and hardware
  • ‚Ä¢ Zero Cost: No development fees, no distribution costs, no licensing requirements
