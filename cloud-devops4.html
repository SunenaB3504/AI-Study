üèóÔ∏è Cloud Computing Fundamentals

üîç Cloud Service Models

üè¢ Infrastructure as a Service (IaaS)

Provides virtualized computing resources over the internet. You manage applications, data, runtime, middleware, and OS.

‚öôÔ∏è Platform as a Service (PaaS)

Provides a platform allowing customers to develop, run, and manage applications without dealing with infrastructure.

üì± Software as a Service (SaaS)

Software that's hosted and managed by the provider. Users access it via web browsers or APIs.

üåê Cloud Deployment Models

üè¢ Public Cloud

Services offered by third-party providers over the public internet. Resources are shared among multiple organizations.

üè† Private Cloud

Cloud infrastructure operated solely for a single organization. Can be managed internally or by a third party.

üîÑ Hybrid Cloud

Combination of public and private clouds, allowing data and applications to be shared between them.

‚òÅÔ∏è Multi-Cloud

Using multiple cloud providers to avoid vendor lock-in and optimize costs and performance.

üîÑ Multi-Cloud Strategies

üåê Multi-Cloud Benefits

üí™ Advantages of Multi-Cloud:

  • Avoid Vendor Lock-in: Not dependent on a single provider
  • Cost Optimization: Choose best pricing for each service
  • Disaster Recovery: Geographic redundancy across providers
  • Best-of-Breed Services: Use strongest service from each provider
  • Risk Mitigation: Reduce impact of provider outages
  • Compliance: Meet regulatory requirements across regions

üõ°Ô∏è Security Best Practices

üîê Identity & Access Management

üõ°Ô∏è Security Principles:

  • Least Privilege: Grant minimum required permissions
  • Multi-Factor Authentication: Enable MFA for all accounts
  • Role-Based Access Control: Use RBAC for granular permissions

üöÄ Migration Strategies

üìã Migration Approaches

üèóÔ∏è 6R Migration Strategy

üîÑ Migration Options:

  • Rehost: Lift and shift - Move applications as-is
  • Replatform: Make minimal changes for cloud optimization
  • Refactor: Modernize applications for cloud-native features
  • Rearchitect: Rebuild applications for cloud scalability
  • Rebuild: Complete rewrite using cloud-native services
  • Retire: Decommission applications no longer needed
