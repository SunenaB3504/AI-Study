Chapter 3: CSS Fundamentals

Styling Web Pages with Cascading Style Sheets

3.1 Introduction to CSS

CSS (Cascading Style Sheets) is a style sheet language used to describe the presentation and formatting of HTML documents. CSS controls the visual appearance of web pages, including layout, colors, fonts, and animations.

Key Concept: Separation of Concerns

CSS separates content (HTML) from presentation (styling), making websites more maintainable, accessible, and flexible.

CSS Evolution

  • CSS1 (1996) - Basic styling capabilities
  • CSS2 (1998) - Positioning, media types, aural styles
  • CSS2.1 (2004) - Bug fixes and clarifications
  • CSS3 (2011+) - Modules, animations, flexbox, grid

How CSS Works

CSS uses a cascade system where multiple style rules can apply to the same element. The cascade determines which styles take precedence based on:

  1. Importance (normal vs !important)
  2. Specificity (how specific the selector is)
  3. Source order (later rules override earlier ones)

3.2 CSS Syntax and Structure

CSS rules consist of selectors and declarations that define how elements should be styled.

Basic CSS Rule Structure

selector {
    property: value;
    property: value;
}

CSS Comments

/* This is a single-line comment */

/*
This is a multi-line comment
that spans several lines
*/

Adding CSS to HTML

There are three ways to add CSS to HTML documents:

1. External Stylesheet (Recommended)

<!-- HTML file -->
<head>
    <link rel="stylesheet" href="styles.css">
</head>

2. Internal Stylesheet

<!-- HTML file -->
<head>
    <style>
        body { background-color: lightblue; }
    </style>
</head>

3. Inline Styles

<!-- HTML file -->
<p style="color: red; font-size: 20px;">This is red text</p>

3.3 CSS Selectors

Selectors target HTML elements to apply styles. CSS provides various types of selectors for different targeting needs.

Basic Selectors

Selector Example Description
Element p { } Selects all paragraph elements
Class .classname { } Selects elements with class="classname"
ID #idname { } Selects element with id="idname"
Universal * { } Selects all elements

Combinator Selectors

Selector Example Description
Descendant div p { } Paragraphs inside div elements
Child div > p { } Direct child paragraphs of div
Adjacent Sibling h1 + p { } Paragraphs immediately after h1
General Sibling h1 ~ p { } All paragraphs after h1

Attribute Selectors

/* Selects all input elements with type="text" */
input[type="text"] { }

/* Selects all elements with href attribute */
a[href] { }

/* Selects links that start with "https" */
a[href^="https"] { }

/* Selects links that end with ".pdf" */
a[href$=".pdf"] { }

/* Selects links containing "example" */
a[href*="example"] { }

Pseudo-Classes and Pseudo-Elements

/* Pseudo-classes */
a:hover { color: red; }
input:focus { border: 2px solid blue; }
li:first-child { font-weight: bold; }
li:last-child { margin-bottom: 0; }

/* Pseudo-elements */
p::first-line { font-weight: bold; }
p::first-letter { font-size: 2em; }
p::before { content: "→ "; }
p::after { content: " ←"; }

3.4 The CSS Box Model

Every HTML element is treated as a rectangular box with content, padding, border, and margin.

Box Model Components

  • Content - The actual content (text, images, etc.)
  • Padding - Space between content and border
  • Border - Line around the padding
  • Margin - Space outside the border

Box Model Properties

.box {
    /* Content dimensions */
    width: 200px;
    height: 100px;

    /* Padding */
    padding-top: 10px;
    padding-right: 20px;
    padding-bottom: 10px;
    padding-left: 20px;

    /* Shorthand: top right bottom left */
    padding: 10px 20px 10px 20px;

    /* Border */
    border-width: 2px;
    border-style: solid;
    border-color: black;

    /* Shorthand */
    border: 2px solid black;

    /* Margin */
    margin: 10px;
}

3.5 Colors and Units

Color Values

CSS supports various ways to specify colors:

/* Named colors */
color: red;
color: blue;
color: green;

/* Hexadecimal */
color: #FF0000;    /* Red */
color: #00FF00;    /* Green */
color: #0000FF;    /* Blue */
color: #FFFFFF;    /* White */
color: #000000;    /* Black */

/* RGB */
color: rgb(255, 0, 0);     /* Red */
color: rgb(0, 255, 0);     /* Green */
color: rgb(0, 0, 255);     /* Blue */

/* RGBA (with alpha) */
color: rgba(255, 0, 0, 0.5);  /* Semi-transparent red */

/* HSL */
color: hsl(0, 100%, 50%);     /* Red */
color: hsl(120, 100%, 50%);   /* Green */
color: hsl(240, 100%, 50%);   /* Blue */

CSS Units

Unit Type Unit Description Example
Absolute px Pixels width: 200px;
pt Points (1/72 inch) font-size: 12pt;
cm Centimeters width: 10cm;
Relative % Percentage width: 50%;
em Relative to font-size font-size: 1.2em;
rem Relative to root font-size font-size: 1.2rem;
vw/vh Viewport width/height width: 50vw;

3.6 Typography

CSS provides extensive control over text appearance and formatting.

Font Properties

.text-example {
    /* Font family */
    font-family: 'Arial', sans-serif;

    /* Font size */
    font-size: 16px;

    /* Font weight */
    font-weight: bold;  /* or 100-900 */

    /* Font style */
    font-style: italic;

    /* Text decoration */
    text-decoration: underline;

    /* Text transformation */
    text-transform: uppercase;

    /* Letter spacing */
    letter-spacing: 2px;

    /* Line height */
    line-height: 1.5;

    /* Text alignment */
    text-align: center;
}

Web Fonts

Use Google Fonts or custom fonts with @font-face:

/* Google Fonts */
@import url('https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap');

body {
    font-family: 'Roboto', sans-serif;
}

/* Custom font with @font-face */
@font-face {
    font-family: 'MyCustomFont';
    src: url('fonts/custom-font.woff2') format('woff2'),
         url('fonts/custom-font.woff') format('woff');
}

3.7 Layout and Positioning

Display Property

/* Block elements */
div { display: block; }

/* Inline elements */
span { display: inline; }

/* Inline-block */
.button { display: inline-block; }

/* Hide elements */
.hidden { display: none; }

Position Property

/* Static (default) */
.static { position: static; }

/* Relative */
.relative {
    position: relative;
    top: 10px;
    left: 20px;
}

/* Absolute */
.absolute {
    position: absolute;
    top: 0;
    right: 0;
}

/* Fixed */
.fixed {
    position: fixed;
    bottom: 0;
    left: 0;
}

/* Sticky */
.sticky {
    position: sticky;
    top: 0;
}

Float and Clear

/* Float elements */
.left-float { float: left; }
.right-float { float: right; }

/* Clear floats */
.clearfix::after {
    content: "";
    clear: both;
    display: table;
}

3.8 Responsive Design

Create websites that work on all devices using media queries and flexible layouts.

Media Queries

/* Mobile first approach */
.container {
    width: 100%;
}

/* Tablet */
@media (min-width: 768px) {
    .container {
        width: 750px;
    }
}

/* Desktop */
@media (min-width: 992px) {
    .container {
        width: 970px;
    }
}

/* Large desktop */
@media (min-width: 1200px) {
    .container {
        width: 1170px;
    }
}

Flexible Images

img {
    max-width: 100%;
    height: auto;
}

/* Or use responsive images */
<img src="small.jpg"
     srcset="medium.jpg 768w, large.jpg 1200w"
     sizes="(max-width: 768px) 100vw, 50vw"
     alt="Responsive image">

3.9 Flexbox Layout

Flexbox provides a modern way to create flexible layouts that adapt to different screen sizes.

Flex Container Properties

.flex-container {
    display: flex;

    /* Direction */
    flex-direction: row; /* row | row-reverse | column | column-reverse */

    /* Wrapping */
    flex-wrap: wrap; /* nowrap | wrap | wrap-reverse */

    /* Shorthand */
    flex-flow: row wrap;

    /* Alignment */
    justify-content: space-between; /* flex-start | flex-end | center | space-between | space-around */
    align-items: center; /* flex-start | flex-end | center | baseline | stretch */
    align-content: flex-start; /* flex-start | flex-end | center | space-between | space-around | stretch */
}

Flex Item Properties

.flex-item {
    /* Growth */
    flex-grow: 1; /* How much it grows */

    /* Shrink */
    flex-shrink: 1; /* How much it shrinks */

    /* Basis */
    flex-basis: auto; /* Initial size */

    /* Shorthand */
    flex: 1 1 auto;

    /* Self alignment */
    align-self: flex-start; /* auto | flex-start | flex-end | center | baseline | stretch */
}

3.10 CSS Grid Layout

CSS Grid provides two-dimensional layout capabilities for complex web layouts.

Grid Container

.grid-container {
    display: grid;

    /* Define columns */
    grid-template-columns: 1fr 2fr 1fr;

    /* Define rows */
    grid-template-rows: auto 1fr auto;

    /* Gap between items */
    gap: 20px;

    /* Alternative syntax */
    grid-template-columns: repeat(3, 1fr);
    grid-template-columns: 200px 1fr 200px;
}

Grid Items

.grid-item {
    /* Position items */
    grid-column: 1 / 3; /* Start at line 1, end at line 3 */
    grid-row: 2 / 4; /* Start at line 2, end at line 4 */

    /* Shorthand */
    grid-area: 2 / 1 / 4 / 3;
}

3.11 CSS Animations and Transitions

Add smooth transitions and animations to enhance user experience.

CSS Transitions

.button {
    background-color: blue;
    transition: background-color 0.3s ease;
}

.button:hover {
    background-color: red;
}

/* Multiple properties */
.element {
    transition: all 0.5s cubic-bezier(0.4, 0, 0.2, 1);
}

CSS Animations

@keyframes slideIn {
    from {
        transform: translateX(-100%);
        opacity: 0;
    }
    to {
        transform: translateX(0);
        opacity: 1;
    }
}

.animated-element {
    animation: slideIn 0.5s ease-out;
}

/* Multiple keyframes */
@keyframes bounce {
    0%, 20%, 50%, 80%, 100% {
        transform: translateY(0);
    }
    40% {
        transform: translateY(-30px);
    }
    60% {
        transform: translateY(-15px);
    }
}

3.12 CSS Best Practices

Organization and Maintainability

  • Use consistent naming conventions (BEM, SMACSS)
  • Organize CSS into logical sections
  • Use CSS variables for reusable values
  • Avoid !important when possible
  • Comment your CSS code

Performance

  • Minimize CSS file size
  • Use CSS sprites for small images
  • Avoid expensive selectors
  • Use hardware acceleration when possible
  • Load critical CSS first

Accessibility

  • Ensure sufficient color contrast
  • Don't rely on color alone for information
  • Respect user's motion preferences
  • Provide focus indicators
  • Test with screen readers

3.13 Summary

CSS is essential for creating visually appealing and responsive web pages:

Key Takeaways

  • CSS separates presentation from content
  • The cascade determines which styles apply
  • The box model controls element dimensions
  • Selectors target specific HTML elements
  • Flexbox and Grid provide modern layout options
  • Media queries enable responsive design
  • Animations enhance user experience
  • Best practices ensure maintainable and performant CSS

Practice Exercises

  2. Build a card layout using Flexbox

  4. Implement smooth transitions and animations
  5. Create a mobile-first responsive design
