Introduction to Android

Android is an open source software stack that includes the operating system, middleware, and key applications along with a set of API libraries for writing mobile applications that can shape the look, feel, and function of mobile handsets.

What Android Is: A comprehensive platform for mobile devices, all of the software to run a mobile phone but without the proprietary obstacles that have hindered mobile innovation.

A Little Background

Mobile phones have evolved from simple communication devices to powerful multifunctional tools incorporating cameras, media players, GPS systems, and touch screens. However, the software platforms have struggled to keep pace with this hardware evolution.

The Not So Distant Past

Historically, mobile development was limited by proprietary operating systems and closed environments that restricted third-party applications. Java MIDlets provided some improvement but still offered limited hardware access.

The Future

Android represents a new wave of mobile operating systems designed for increasingly powerful mobile hardware. It provides an open development environment built on an open source Linux kernel with total access to device hardware and capabilities.

What Android Isn't

  • Not a Java ME implementation: Android applications are written in Java but run on Dalvik VM, not Java ME
  • Not part of LiPS or OMA: While goals are similar, Android provides a complete software stack
  • Not just an application layer: Android includes the complete software stack from OS to applications
  • Not a single handset: Android is a platform that can run on many different devices
  • Not iPhone's answer: Android is open source and equal, not proprietary like iPhone

An Open Platform for Mobile Development

Android consists of several essential and dependent parts:

  • Hardware reference design - Describes capabilities required for mobile devices
  • Linux operating system kernel - Provides low-level interface with hardware
  • Open source libraries - Including SQLite, WebKit, OpenGL, and media manager
  • Android run time - Includes core libraries and Dalvik virtual machine
  • Application framework - Provides services for and management of applications
  • User interface framework - Hosts and launches applications
  • Preinstalled applications - Shipped as part of the stack
  • Software development kit - Tools, APIs, documentation for creating applications

Understanding the Android Software Stack

Applications
Application Framework
Android Runtime
Libraries
Linux Kernel

Linux Kernel

Core services including hardware drivers, process and memory management, security, network, and power management - all optimized for mobile devices.

Libraries

C/C++ core libraries including libc, SSL, media libraries, graphics libraries (SGL, OpenGL), SQLite, and WebKit.

Android Runtime

Core libraries and Dalvik virtual machine that power applications and form the basis for the application framework.

Application Framework

Classes used to create Android applications, providing services for window management, content providers, location services, telephony, and peer-to-peer services.

Application Layer

All applications, both native and third-party, built on the same API libraries and executed on the same run time.

The Dalvik Virtual Machine

Dalvik is Android's custom virtual machine optimized for mobile devices:

  • Register-based architecture - Optimized for efficiency
  • Multiple instance support - Runs multiple VMs efficiently
  • Linux kernel integration - Uses kernel for threading and memory management
  • Dex format - Optimized executable format for minimal memory footprint

Android Application Architecture

Android encourages component reuse through loosely coupled architecture:

Activities

Application's presentation layer - every screen is an Activity extension

Services

Invisible workers for background processing and data updates

Content Providers

Shareable data stores for inter-application data sharing

Intents

Message-passing framework for component communication

Broadcast Receivers

Intent broadcast consumers for event-driven applications

Notifications

User notification framework for background services

Advanced Android Libraries

Additional APIs for advanced functionality:

  • android.location - Location-based services using GPS and cell towers
  • android.media - Audio and video playback and recording
  • android.opengl - 3D graphics with OpenGL ES
  • Hardware APIs - Camera, accelerometer, compass sensors
  • Communication APIs - Bluetooth, Wi-Fi, telephony

Why Develop for Android?

Android offers developers the opportunity to create innovative software applications for mobile devices without the restrictions generally associated with existing proprietary mobile development frameworks.

Key Advantages:

  • Open platform - No artificial barriers for developers
  • Equal access - Same APIs for native and third-party applications
  • Hardware freedom - Full access to device capabilities
  • Rich APIs - Comprehensive libraries for advanced functionality
  • Thriving community - Active developer community and support
  • No licensing fees - Free development and distribution

Practice Exercise: Setting Up Your Development Environment

  1. Download the Android SDK from the official Android website
  2. Install Java Development Kit (JDK) version 5 or 6
  3. Download and install Eclipse IDE
  4. Install the Android Development Tools (ADT) plug-in for Eclipse
  5. Configure the ADT plug-in with your SDK location
  6. Create your first Android project using the New Project Wizard
  7. Run the Hello World application on the Android emulator
