Section 1: Android Networking Fundamentals

Android provides several ways to handle network operations: HttpURLConnection, Volley, Retrofit, and OkHttp. Always perform network operations on background threads to avoid blocking the UI.

Available Networking Libraries:

  • ‚Ä¢ HttpURLConnection: Built-in Android networking
  • ‚Ä¢ Volley: Google's lightweight networking library
  • ‚Ä¢ Retrofit: Type-safe HTTP client (most popular)
  • ‚Ä¢ OkHttp: HTTP client with advanced features

‚ö†Ô∏è Important Note:

Network operations must never be performed on the main UI thread. Always use background threads or AsyncTask alternatives to prevent ANR (Application Not Responding) errors.

Section 3: HTTP Requests with HttpURLConnection

public class NetworkUtils {
public static String makeHttpRequest(URL url) throws IOException {
HttpURLConnection connection = null;
InputStream inputStream = null;

try {
connection = (HttpURLConnection) url.openConnection();
connection.setRequestMethod("GET");
connection.setReadTimeout(10000);
connection.setConnectTimeout(15000);
connection.connect();

int responseCode = connection.getResponseCode();
if (responseCode != 200) {
throw new IOException("HTTP error code: " + responseCode);
}

inputStream = connection.getInputStream();
return readFromStream(inputStream);

} finally {
if (connection != null) {
connection.disconnect();
}
if (inputStream != null) {
inputStream.close();
}
}
}

private static String readFromStream(InputStream inputStream) throws IOException {
StringBuilder output = new StringBuilder();
if (inputStream != null) {
InputStreamReader inputStreamReader = new InputStreamReader(inputStream, Charset.forName("UTF-8"));
BufferedReader reader = new BufferedReader(inputStreamReader);
String line = reader.readLine();
while (line != null) {
output.append(line);
line = reader.readLine();
}
}
return output.toString();
}
}

HttpURLConnection Best Practices:

  • ‚Ä¢ Always set connection and read timeouts
  • ‚Ä¢ Properly close streams and connections
  • ‚Ä¢ Handle HTTP error codes appropriately
  • ‚Ä¢ Use try-with-resources for automatic cleanup
  • ‚Ä¢ Check network availability before making requests

// Add to build.gradle
dependencies {

// Simple GET request
StringRequest stringRequest = new StringRequest(Request.Method.GET, url,
new Response.Listener<String>() {
@Override
public void onResponse(String response) {
// Handle response
Log.d("Response", response);
}
},
new Response.ErrorListener() {
@Override
public void onErrorResponse(VolleyError error) {
// Handle error

}

// Add request to queue
Volley.newRequestQueue(this).add(stringRequest);

Section 6: Network Security & Best Practices

Security Best Practices:

  • ‚Ä¢ Always use HTTPS in production
  • ‚Ä¢ Implement SSL certificate pinning
  • ‚Ä¢ Validate all input data
  • ‚Ä¢ Handle network errors gracefully
  • ‚Ä¢ Use appropriate timeouts
  • ‚Ä¢ Implement proper authentication
// Network state checking
public static boolean isNetworkAvailable(Context context) {
ConnectivityManager cm = (ConnectivityManager) context
.getSystemService(Context.CONNECTIVITY_SERVICE);
NetworkInfo activeNetwork = cm.getActiveNetworkInfo();
return activeNetwork != null && activeNetwork.isConnected();
}

Threading Guidelines:

  • ‚Ä¢ Never perform network operations on main thread
  • ‚Ä¢ Use background threads or modern alternatives
  • ‚Ä¢ Handle configuration changes properly
  • ‚Ä¢ Cancel requests when appropriate
  • ‚Ä¢ Show loading indicators to users
  • ‚Ä¢ AsyncTask is deprecated - use modern approaches

Recommended Libraries:

For modern Android development, prefer Retrofit with OkHttp for most networking needs. Use WorkManager for background tasks and consider Kotlin Coroutines for asynchronous programming.
