Chapter 3: App Manifest

üìã What is the Web App Manifest?

The Web App Manifest is a JSON file that provides information about your Progressive Web App (PWA) to the browser. It defines how your app appears when installed on a user's device and controls the app's behavior.

üìÑ Basic Manifest Structure

Here's a basic example of a web app manifest:

{
    "name": "My Progressive Web App",
    "short_name": "MyPWA",
    "description": "A sample progressive web app",
    "start_url": "/",
    "display": "standalone",
    "background_color": "#ffffff",
    "theme_color": "#4285f4"
}

üè∑Ô∏è Essential Properties

  • name - The full name of your app
  • short_name - A shorter version for limited space
  • start_url - The URL that loads when launched
  • display - How the app appears (standalone, fullscreen, etc.)
  • icons - App icons for different sizes

üì± Display Modes

The display property controls how your PWA appears:

  • standalone - Opens like a native app
  • fullscreen - Full screen mode
  • minimal-ui - Minimal browser UI
  • browser - Normal browser mode

üé® Icons and Colors

Icons are crucial for your PWA's appearance:

  • 192x192 and 512x512 pixels recommended
  • PNG format preferred
  • Use maskable icons for adaptive shapes

‚úÖ Testing Your Manifest

To test your manifest:

  1. Validate JSON syntax
  2. Test on different devices
  3. Check installation prompts
  4. Verify icons display correctly
