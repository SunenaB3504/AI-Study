2.1 What You Need to Begin

Android applications run on the Android Runtime (ART), which supports modern development platforms. As of 2025, Android development is supported on:

  • Microsoft Windows (10 or later, 64-bit)
  • macOS (10.14 or later)
  • Linux (Ubuntu 18.04 or later, 64-bit)
  • Chrome OS (with Linux support)

To get started, you'll need to download and install:

  • Android Studio (the official IDE)
  • Java Development Kit (JDK) 11 or higher (included with Android Studio)
  • Android SDK (automatically managed by Android Studio)

Android Studio includes everything you need for Android development, including the latest SDK tools, emulator, and build system. You can download it from the official Android Developer website at https://developer.android.com/studio

The Android platform supports multiple programming languages, with Kotlin as the preferred language for new projects, though Java is still fully supported.

2.2 Downloading and Installing Android Studio

Android Studio is Google's official IDE for Android development and is completely free. It includes the Android SDK, build tools, emulator, and all necessary components. You can download the latest version from:

https://developer.android.com/studio

As of 2025, the current stable version is Android Studio Jellyfish (2023.3.1) or later, with support for Android 15 (API level 35).

The installation package includes:

  • Android SDK with the latest platform tools
  • Android Emulator with hardware acceleration support
  • Gradle build system for project compilation
  • Integrated development environment with code completion, debugging, and profiling
  • SDK Manager for downloading additional components
  • AVD Manager for creating virtual devices

After installation, Android Studio will automatically download the latest SDK components. You can also use any text editor or IDE you're comfortable with, but Android Studio provides the most integrated experience with features like:

  • Real-time code analysis and suggestions
  • Built-in emulator with various device configurations
  • Layout editor for UI design
  • APK analyzer and profiler
  • Version control integration (Git)

2.3 Setting Up Android Studio

Android Studio provides a complete development environment optimized for Android development.

After downloading and installing Android Studio from the official website, launch it to complete the setup:

  1. Android Studio will automatically download the latest Android SDK and build tools
  2. Configure the SDK location (usually in your user folder under Android/sdk)
  3. Install any additional SDK components you need (different API levels, system images)
  4. Create your first project using the New Project wizard

Android Studio includes:

  • Intelligent code editor with Kotlin and Java support
  • Visual layout editor for designing UIs
  • Built-in terminal and version control
  • Device Manager for creating and managing emulators
  • Profiler for performance analysis
  • APK Analyzer for inspecting app packages

The IDE is based on IntelliJ IDEA and provides excellent support for Android-specific features like resource management, manifest editing, and debugging.

2.4 Android Studio Features

Android Studio provides comprehensive tools for Android development, all integrated into a single IDE. Key features include:

  • Project Templates: Pre-built templates for different app types (Empty Activity, Basic Activity, Navigation Drawer, etc.)
  • Layout Editor: Drag-and-drop UI designer with real-time preview on multiple device sizes
  • Code Editor: Advanced editing with code completion, refactoring, and real-time error detection for both Kotlin and Java
  • Emulator: High-performance Android Emulator with hardware acceleration and various device configurations
  • Device Manager: Create and manage virtual devices for testing
  • SDK Manager: Download and manage SDK components, platforms, and tools
  • Build System: Gradle-based build system with support for variants, flavors, and custom build configurations
  • Debugging: Advanced debugging with breakpoints, variable inspection, and device file access
  • Profiling: CPU, memory, and network profiling tools
  • Version Control: Integrated Git support with commit, push, and merge capabilities

Android Studio also includes the Android Debug Bridge (ADB) for device communication and the Dalvik Debug Monitor Server (DDMS) for advanced debugging and monitoring.

2.5 Setting Up Your Development Environment

Setting up Android Studio is straightforward:

  1. Download and Install: Download Android Studio from https://developer.android.com/studio and run the installer
  2. Initial Setup: Launch Android Studio and follow the setup wizard
  3. SDK Installation: Android Studio will automatically download the latest Android SDK and build tools
  4. Configure SDK: The SDK location is typically ~/Android/Sdk on macOS/Linux or C:\Users\[username]\AppData\Local\Android\Sdk on Windows
  5. Install Additional Components: Use the SDK Manager to download additional platforms, system images, and tools as needed
  6. Create AVD: Use the Device Manager to create Android Virtual Devices for testing
  7. Enable Hardware Acceleration: Install Intel HAXM or AMD Hypervisor for faster emulator performance

Android Studio includes everything needed for development. The setup wizard will guide you through installing any missing components.

2.6 Creating Your First Android Project

You've installed Android Studio and set up the SDK. You're now ready to create your first Android project.

Starting a New Android Project

To create a new Android project in Android Studio:

  1. Launch Android Studio
  2. Select "New Project" from the welcome screen or File → New → New Project
  3. Choose a project template (e.g., "Empty Activity" for a basic app)
  4. Configure your project:
    • Name: Your app's name
    • Package name: Unique identifier (e.g., com.example.myapp)
    • Save location: Where to store the project
    • Language: Kotlin (recommended) or Java
    • Minimum SDK: The lowest Android version your app supports
  5. Click "Finish" to create the project

Android Studio will create a complete project structure with:

  • Source code directories (java/kotlin for code, res for resources)
  • Build configuration files (build.gradle)
  • Android Manifest (AndroidManifest.xml)
  • Default activity and layout files

The default template creates a simple "Hello World" app that you can run immediately to test your setup.

2.7 Running and Debugging Your App

Android Studio makes running and debugging your app straightforward. You can run your app on:

  • Android Emulator: Virtual devices that simulate various Android devices
  • Physical Devices: Connected Android phones/tablets via USB

Setting Up Run Configurations

To configure how your app runs:

  1. Select Run → Edit Configurations
  2. Choose your app module
  3. Configure options like:
    • Deployment target (emulator or connected device)
    • Launch options (default activity, URL, etc.)
    • Debugger settings

Creating Virtual Devices

Use the Device Manager to create AVDs (Android Virtual Devices):

  1. Open Device Manager (Tools → Device Manager)
  2. Click "Create device"
  3. Choose a device definition (Pixel, Nexus, etc.)
  4. Select a system image (Android version)
  5. Configure hardware properties
  6. Click "Finish"

Once configured, you can run your app by clicking the green "Run" button or pressing Shift+F10.

2.8 Building and Testing Your Android Applications

Android Studio uses Gradle as its build system, which provides powerful build capabilities including:

  • Automatic dependency management
  • Build variants (debug/release)
  • Product flavors for different app versions
  • Custom build tasks

Running Your App

To run your app:

  1. Connect a physical device or start an emulator
  2. Click the "Run" button (green play icon) or press Shift+F10
  3. Select your target device
  4. Android Studio will:
    • Compile your code
    • Build the APK
    • Install it on the device/emulator
    • Launch the app

Debugging

Android Studio provides advanced debugging features:

  • Set breakpoints by clicking in the gutter
  • Use the Debug button (Shift+F9) to start debugging
  • Inspect variables and expressions
  • Step through code execution
  • View device logs in Logcat
  • Use the debugger for network inspection and database viewing

The build process converts your Kotlin/Java code to DEX bytecode and packages everything into an APK file ready for distribution.

2.9 Understanding Your First Android App

Let's examine the structure of your first Android application created by Android Studio.

Activity Structure

Activities are the building blocks of Android applications. Here's a modern Kotlin implementation:

package com.example.myfirstapp

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
    }
}

Key differences from the old Java version:

  • Uses Kotlin syntax with null safety
  • Extends AppCompatActivity for backward compatibility
  • Uses modern AndroidX libraries
  • onCreate parameter is nullable (Bundle?)

Layout Files

Modern Android uses ConstraintLayout for flexible UI design:

<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintLeft_toLeftOf="parent"
        app:layout_constraintRight_toRightOf="parent"
        app:layout_constraintTop_toTopOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>

Modern Android development emphasizes:

  • Separation of concerns with ViewModels and LiveData
  • Declarative UI with Jetpack Compose (optional)
  • Material Design 3 components
  • Modern app architecture patterns

2.10 Types of Android Applications

Most of the applications you create in Android will fall into one of the following categories:

Foreground Activity

An application that's only useful when it's in the foreground and is effectively suspended when it's not visible. Games and map mashups are common examples.

Background Service

An application with limited interaction that, apart from when being configured, spends most of its lifetime hidden. Examples of this include call screening applications or SMS auto-responders.

Intermittent Activity

Expects some interactivity but does most of its work in the background. Often these applications will be set up and then run silently, notifying users when appropriate. A common example would be a media player.

Complex applications are difficult to pigeonhole into a single category and include elements of all three. When creating your application, you need to consider how it's likely to be used and then design it accordingly.

2.11 Developing for Modern Android Devices

Android development has evolved significantly since 2008. Modern Android devices include diverse form factors and capabilities that require careful consideration.

Modern Hardware Considerations

Today's Android devices offer:

  • High-resolution displays (up to 4K)
  • Foldable and dual-screen devices
  • 5G connectivity with low latency
  • Advanced cameras and sensors
  • Powerful processors with AI capabilities
  • Large storage capacities
  • Improved battery life and fast charging

Key Development Principles

Modern Android development emphasizes:

  • Responsive Design: Support for multiple screen sizes and orientations
  • Material Design: Consistent, beautiful UI following Material Design 3
  • Privacy-First: Respect user data with proper permissions and transparency
  • Performance: Optimize for battery life and smooth user experience
  • Accessibility: Make apps usable by everyone
  • Security: Use modern security best practices
  • Modularity: Build with reusable components and libraries

Architecture Patterns

Modern apps use:

  • MVVM (Model-View-ViewModel) with LiveData and ViewModels
  • Repository pattern for data management
  • Dependency injection with Hilt or Dagger
  • Coroutines for asynchronous programming
  • Jetpack Compose for declarative UI (optional)

Always test on a variety of devices and use tools like Firebase Test Lab for automated testing across device configurations.
