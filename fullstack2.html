Chapter 2: HTML Fundamentals

Building the Structure of Web Pages

2.1 Introduction to HTML

HTML (HyperText Markup Language) is the backbone of every web page. As a markup language, HTML defines the structure and semantics of web content using elements that tell browsers how to display information.

Key Concept: Markup Language

HTML is not a programming language but a markup language that uses tags to annotate text, images, and other content for display in web browsers.

HTML5: The Modern Standard

HTML5, released in 2014 and continuously evolving, introduced semantic elements, multimedia support, and enhanced APIs for modern web development.

Basic HTML5 Document Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Web Page</title>
</head>
<body>
    <h1>Hello World</h1>
    <p>This is my first web page.</p>
</body>
</html>

2.2 HTML Document Structure

Every HTML document follows a standard structure with required and optional elements.

Essential Elements

  • <!DOCTYPE html> - Document type declaration
  • <html> - Root element
  • <head> - Document metadata
  • <body> - Document content

Semantic HTML5 Elements

HTML5 introduced semantic elements that provide meaning to content:

Document Structure

  • <header> - Page or section header
  • <nav> - Navigation links
  • <main> - Main content
  • <article> - Self-contained content
  • <section> - Thematic grouping
  • <aside> - Sidebar content
  • <footer> - Page or section footer

2.3 Text Formatting Elements

HTML provides various elements for structuring and formatting text content.

Headings

HTML provides six levels of headings, from most important (h1) to least important (h6):

<h1>Main Heading (Page Title)</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Least Important Heading</h6>

Paragraphs and Text Structure

<p>This is a paragraph of text.</p>

<p>You can create <strong>bold text</strong> and <em>emphasized text</em>.</p>

<blockquote>
    <p>This is a blockquote for longer quotations.</p>
</blockquote>

Text-Level Semantics

Element Purpose Example
<strong> Strong importance Bold text
<em> Emphasis Italic text
<mark> Highlighted text Marked text
<small> Small print Fine print
<cite> Title of work Book Title

2.4 Links and Images

Hyperlinks

Links connect web pages and resources. The <a> element creates hyperlinks:

<!-- External link -->
<a href="https://www.example.com">Visit Example</a>

<!-- Internal link -->
<a href="about.html">About Us</a>

<!-- Link with target -->
<a href="document.pdf" target="_blank">Download PDF</a>

<!-- Email link -->
<a href="mailto:contact@example.com">Email Us</a>

Images

The <img> element embeds images. Always include alt text for accessibility:

<!-- Basic image -->
<img src="image.jpg" alt="Description of image">

<!-- Image with dimensions -->
<img src="photo.jpg" alt="Scenic landscape" width="400" height="300">

<!-- Responsive image -->
<img src="responsive.jpg" alt="Responsive design" style="max-width: 100%;">

Accessibility Best Practice

Always provide meaningful alt text for images. For decorative images, use alt="" or CSS background images instead.

2.5 Lists

HTML provides three types of lists for organizing content.

Unordered Lists

Use <ul> for lists where order doesn't matter:

<ul>
    <li>HTML</li>
    <li>CSS</li>
    <li>JavaScript</li>
</ul>

Ordered Lists

Use <ol> for numbered lists where order matters:

<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

Description Lists

Use <dl> for term-definition pairs:

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
</dl>

2.6 Tables

Tables organize data into rows and columns. Use semantic markup for proper accessibility.

Basic Table Structure

<table>
    <caption>Web Development Technologies</caption>
    <thead>
        <tr>
            <th>Technology</th>
            <th>Purpose</th>
            <th>Difficulty</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>HTML</td>
            <td>Structure</td>
            <td>Beginner</td>
        </tr>
        <tr>
            <td>CSS</td>
            <td>Styling</td>
            <td>Intermediate</td>
        </tr>
        <tr>
            <td>JavaScript</td>
            <td>Interactivity</td>
            <td>Advanced</td>
        </tr>
    </tbody>
</table>

Table Elements

  • <table> - Table container
  • <caption> - Table title
  • <thead> - Table header
  • <tbody> - Table body
  • <tfoot> - Table footer
  • <tr> - Table row
  • <th> - Header cell
  • <td> - Data cell

2.7 Forms

Forms collect user input and send it to servers for processing.

Basic Form Structure

<form action="/submit" method="post">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>

    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>

    <button type="submit">Submit</button>
</form>

Common Input Types

Input Type Description Example
text Single line text <input type="text">
email Email address <input type="email">
password Password field <input type="password">
number Numeric input <input type="number">
date Date picker <input type="date">
checkbox Multiple choice <input type="checkbox">
radio Single choice <input type="radio">
submit Form submission <input type="submit">

2.8 HTML Attributes

Attributes provide additional information about HTML elements.

Common Attributes

  • id - Unique identifier for an element
  • class - Class name for styling and JavaScript
  • style - Inline CSS styles
  • title - Tooltip text
  • lang - Language of the element
  • dir - Text direction (ltr/rtl)
<div id="main-content" class="container" lang="en">
    <h1 title="Welcome Message">Hello World</h1>
    <p class="introduction">This is a sample paragraph.</p>
</div>

2.9 Best Practices

Semantic HTML

Use semantic elements to provide meaning and improve accessibility:

❌ Avoid:

<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>

✅ Use:

<header>...</header>
<nav>...</nav>
<main>...</main>

Accessibility

  • Always use alt text for images
  • Provide labels for form inputs
  • Use semantic headings (h1-h6)
  • Ensure sufficient color contrast
  • Make interactive elements keyboard accessible

Performance

  • Use appropriate image formats and compression
  • Specify image dimensions
  • Minimize DOM depth
  • Use semantic HTML for better SEO

2.10 Summary

In this chapter, we covered the fundamentals of HTML:

Key Takeaways

  • HTML provides the structure and semantics of web pages
  • HTML5 introduced semantic elements and enhanced multimedia support
  • Proper document structure is essential for accessibility and SEO
  • Semantic HTML improves code maintainability and accessibility
  • Forms collect user input with various input types
  • Attributes provide additional information about elements
  • Best practices ensure accessible, performant, and maintainable code
